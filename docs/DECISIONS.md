# Decisions

## 2026-06-30：V1 使用 API-only 只读路线

决定：V1 只通过蓝湖 Web API 读取项目和设计稿信息，不做浏览器自动化兜底。

原因：

- 浏览器拖拽画布、截图和 OCR 路径笨重且不稳定。
- API-only 更适合 MCP 工具化，也更容易测试。
- 失败时返回结构化错误和建议，比隐式浏览器兜底更可控。

影响：

- 遇到 403/418 或接口字段变化时，V1 不自动打开浏览器补救。
- 后续优先增强 API 适配和错误提示。

## 2026-06-30：认证只使用 LANHU_COOKIE 环境变量

决定：V1 只读取 `LANHU_COOKIE` 环境变量。

原因：

- 不读取浏览器 Cookie、LocalStorage 或用户私密配置。
- 不保存、不打印 Cookie，安全边界清晰。
- 方便 Codex、Claude、Cursor 等 MCP 客户端配置。

影响：

- 新用户需要手动从蓝湖请求中复制 Cookie。
- Cookie 过期时需要手动更新。

## 2026-06-30：项目内私有 MCP 包

决定：当前阶段不发布 npm 包，先作为项目内私有工具开发。

原因：

- 先验证蓝湖 API 和 Codex 工作流。
- 减少发布、版本兼容和用户支持成本。
- 更适合快速迭代。

影响：

- 使用者需要 clone 仓库、安装依赖、构建后接入 MCP。
- 后续若多人稳定使用，再规划 npm 发布。

## 2026-06-30：多 Agent 协作靠仓库上下文承接

决定：不同 AI Agent 不依赖聊天记录共享上下文，而是依赖仓库文档和 git 历史接力。

原因：

- 多端、多用户、多 AI 客户端之间无法天然共享对话上下文。
- 文档化的项目记忆更稳定，也更适合代码审查和长期维护。

影响：

- 每个阶段必须更新 `STATUS`、`HANDOFF`，必要时更新 `ROADMAP` 和 `DECISIONS`。
- 新 Agent 必须先读 `AGENTS.md` 和 docs 后再开发。

## 2026-07-01：V1.1 优先建设代码还原上下文

决定：下一阶段优先把 `context.md/context.json` 从画板清单升级为辅助 Codex 还原代码的实现上下文。

原因：

- 项目核心目标是辅助 Codex 根据蓝湖设计文稿进行页面/组件代码还原。
- V1 已证明 API-only 读取链路可行，但输出还不足以指导高质量实现。
- 在尚未获得完整图层树、切图和 Design Tokens 前，可以先通过画板名称、资源路径和轻量规则生成页面角色、流程关系和实现建议。

影响：

- 新增 `src/core/restoration.ts` 作为还原推断层，避免把推断逻辑散落在 MCP 工具层或 Markdown 渲染层。
- `context.json` 新增 `restoration` 结构，后续单图详情、切图和 tokens 都应接入该结构或其后续扩展。
- 页面角色和流程推断必须明确标注为轻量推断，不得伪装成蓝湖官方结构化图层数据。
